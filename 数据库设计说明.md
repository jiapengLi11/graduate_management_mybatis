## 数据库概述

### 数据库信息
- **数据库名称**: `db02`
- **字符集**: `utf8mb4`
- **排序规则**: `utf8mb4_unicode_ci`
- **数据库引擎**: `InnoDB`

### 设计原则
1. **规范化设计**: 遵循数据库第三范式，减少数据冗余
2. **外键约束**: 保证数据一致性和完整性
3. **索引优化**: 为常用查询字段建立索引，提高查询性能
4. **扩展性**: 预留扩展字段，便于后续功能扩展

## 数据库表结构

### 1. 学生信息表（user）

**表名**: `user`

**用途**: 存储研究生的基本信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 学生ID |
| user_account | VARCHAR(50) | | 用户账号 |
| student_no | VARCHAR(20) | UNIQUE | 学号（唯一） |
| real_name | VARCHAR(50) | | 真实姓名 |
| age | INT | | 年龄 |
| gender | VARCHAR(10) | | 性别 |
| email | VARCHAR(100) | | 邮箱 |
| phone | VARCHAR(20) | | 电话 |
| major | VARCHAR(100) | | 专业 |
| advisor | VARCHAR(50) | | 导师 |
| grade | VARCHAR(20) | | 年级 |
| study_status | VARCHAR(20) | | 学习状态 |
| enroll_date | DATE | | 入学日期 |
| graduate_date | DATE | | 毕业日期 |
| gpa | DECIMAL(3,2) | | GPA |
| record_create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 记录创建时间 |

**索引**:
- PRIMARY KEY (`id`)
- UNIQUE KEY (`student_no`)
- INDEX (`major`)

### 5. 课程教师关联表（course_teachers）

**表名**: `course_teachers`

**用途**: 存储课程与教师的关联关系（多对多）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 关联ID |

- `course_id` → `courses.id` (ON DELETE CASCADE)

**唯一约束**:
- UNIQUE KEY (`course_id`, `teacher_id`)

**索引**:
- PRIMARY KEY (`id`)
- INDEX (`course_id`)
- INDEX (`teacher_id`)

## 索引设计

### 主键索引
- 所有表的主键字段都自动创建主键索引

### 唯一索引
- `user.student_no` - 学号唯一索引
- `semesters.semester_code` - 学期代码唯一索引
- `courses.course_no` - 课程号唯一索引
- `teachers.teacher_no` - 教师工号唯一索引

### 普通索引
- `user.major` - 专业索引（用于筛选）
- `user.grade` - 年级索引（用于筛选）
- `courses.course_name` - 课程名称索引（用于模糊查询）
- `courses.course_type` - 课程类型索引（用于筛选）
- `courses.semester_id` - 学期ID索引（用于关联查询）
- `student_courses.student_no` - 学号索引（核心查询字段）
- `student_courses.status` - 状态索引（用于筛选）

### 复合索引
- `student_courses(student_no, course_id)` - 唯一约束索引

## 视图设计

### v_student_courses_detail（学生选课详情视图）

**用途**: 整合学生选课相关的所有信息，便于前端查询展示

**包含字段**:
- 选课信息：`select_id`, `student_no`, `select_status`, `grade`, `select_time`
- 课程信息：`course_id`, `course_no`, `course_name`, `course_type`, `credits`, `theory_hours`, `practice_hours`, `hours_display`
- 学期信息：`semester_name`, `semester_code`
- 上课信息：`class_time`, `classroom`
- 教师信息：`teachers`（多个教师用"，"分隔）
- 总学时：`total_hours`

**创建SQL**:
```sql
CREATE VIEW v_student_courses_detail AS
SELECT 
    sc.id AS select_id,
    sc.student_no,
    sc.status AS select_status,
    sc.grade,
    sc.select_time,
    c.id AS course_id,
    c.course_no,
    c.course_name,
    c.course_type,
    c.credits,
    c.theory_hours,
    c.practice_hours,
    CONCAT(c.theory_hours, '/', c.practice_hours) AS hours_display,
    s.semester_name,
    s.semester_code,
    c.class_time,
    c.classroom,
    GROUP_CONCAT(t.teacher_name ORDER BY ct.is_main_teacher DESC SEPARATOR '，') AS teachers,
    (c.theory_hours + c.practice_hours) AS total_hours
FROM 
    student_courses sc
    INNER JOIN courses c ON sc.course_id = c.id
    LEFT JOIN semesters s ON c.semester_id = s.id
    LEFT JOIN course_teachers ct ON c.id = ct.course_id
    LEFT JOIN teachers t ON ct.teacher_id = t.id
GROUP BY 
    sc.id, sc.student_no, sc.status, sc.grade, sc.select_time,
    c.id, c.course_no, c.course_name, c.course_type, c.credits,
    c.theory_hours, c.practice_hours, s.semester_name, s.semester_code,
    c.class_time, c.classroom;
```

## 常用SQL查询

### 1. 查询学生的所有已选课程
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001'
ORDER BY semester_code DESC, course_no ASC;
```

### 2. 按学期筛选课程
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001' 
  AND semester_code = '2024-2025-1';
```

### 3. 按课程类型筛选
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001' 
  AND course_type = '必修';
```

### 4. 模糊查询课程
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001' 
  AND (course_name LIKE '%数据结构%' OR course_no LIKE '%CS%');
```

### 5. 统计已选课程总数
```sql
SELECT COUNT(*) AS total_courses 
FROM student_courses 
WHERE student_no = 'S2024001';
```

### 6. 统计已修学分总数
```sql
SELECT COALESCE(SUM(c.credits), 0) AS total_credits
FROM student_courses sc
INNER JOIN courses c ON sc.course_id = c.id
WHERE sc.student_no = 'S2024001' 
  AND sc.status IN ('已修读', '已结课');
```

### 7. 统计平均学时
```sql
SELECT COALESCE(AVG(c.theory_hours + c.practice_hours), 0) AS avg_hours
FROM student_courses sc
INNER JOIN courses c ON sc.course_id = c.id
WHERE sc.student_no = 'S2024001';
```

### 8. 查询某学期的所有课程
```sql
SELECT c.*, s.semester_name
FROM courses c
LEFT JOIN semesters s ON c.semester_id = s.id
WHERE s.semester_code = '2024-2025-1';
```

### 9. 查询某课程的所有选课学生
```sql
SELECT u.real_name, u.student_no, sc.status, sc.grade
FROM student_courses sc
INNER JOIN user u ON sc.student_no = u.student_no
WHERE sc.course_id = 1;
```

### 10. 查询某课程的所有授课教师
```sql
SELECT t.teacher_name, t.department, ct.is_main_teacher
FROM course_teachers ct
INNER JOIN teachers t ON ct.teacher_id = t.id
WHERE ct.course_id = 1
ORDER BY ct.is_main_teacher DESC;
```

## 注意事项

1. **数据一致性**
   - 删除课程时，会自动删除相关的选课记录（CASCADE）
   - 删除学期时，相关课程的学期ID会置为NULL（SET NULL）
   - 删除教师时，会自动删除课程教师关联记录（CASCADE）

2. **性能优化**
   - 所有外键字段都建立了索引
   - 常用查询字段都建立了索引
   - 建议使用视图 `v_student_courses_detail` 进行查询，避免多表JOIN

3. **数据扩展**
   - 如需支持更多课程类型，只需在 `course_type` 字段中添加新值
   - 如需支持更多选课状态，只需在 `status` 字段中添加新值

4. **与现有表的关系**
   - `student_courses.student_no` 关联 `user.student_no`（需确保学号一致）

5. **字符编码**
   - 所有表使用 `utf8mb4` 字符集，支持emoji和特殊字符
   - 字符串字段长度根据实际需求设置，避免过长影响性能
