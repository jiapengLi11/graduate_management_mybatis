# 课程管理系统数据库设计说明

## 一、数据库表结构概览

本系统共设计 **5个核心表** 和 **1个查询视图**，用于支持研究生已选课程展示功能。

### 表关系图
```
user (学生表 - 已存在)
  ↓ (通过 student_no 关联)
student_courses (学生选课关联表)
  ↓ (通过 course_id 关联)
courses (课程表)
  ├─→ semesters (学期表)
  └─→ course_teachers (课程教师关联表)
        └─→ teachers (教师表)
```

---

## 二、数据表详细设计

### 1. 学期表（semesters）

**用途**：存储学期信息，用于课程筛选和展示

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 学期ID |
| semester_name | VARCHAR(50) | NOT NULL, UNIQUE | 学期名称，如：2024-2025 学年第 1 学期 |
| semester_code | VARCHAR(20) | NOT NULL, UNIQUE | 学期代码，如：2024-2025-1 |
| start_date | DATE | | 学期开始日期 |
| end_date | DATE | | 学期结束日期 |
| is_current | TINYINT(1) | DEFAULT 0 | 是否为当前学期（0-否，1-是） |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_semester_code`：学期代码索引
- `idx_is_current`：当前学期索引

---

### 2. 教师表（teachers）

**用途**：存储教师基本信息

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 教师ID |
| teacher_no | VARCHAR(20) | NOT NULL, UNIQUE | 教师工号 |
| teacher_name | VARCHAR(50) | NOT NULL | 教师姓名 |
| department | VARCHAR(100) | | 所属学院/部门 |
| title | VARCHAR(50) | | 职称，如：教授、副教授、讲师 |
| email | VARCHAR(100) | | 邮箱 |
| phone | VARCHAR(20) | | 联系电话 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `idx_teacher_no`：教师工号索引
- `idx_teacher_name`：教师姓名索引

---

### 3. 课程表（courses）

**用途**：存储课程基本信息，核心表

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 课程ID |
| course_no | VARCHAR(20) | NOT NULL, UNIQUE | 课程号，唯一标识 |
| course_name | VARCHAR(200) | NOT NULL | 课程名称 |
| course_type | VARCHAR(20) | NOT NULL | 课程类型：必修、选修、专业核心、公共基础 |
| credits | DECIMAL(3,1) | NOT NULL | 学分 |
| theory_hours | INT | DEFAULT 0 | 理论学时 |
| practice_hours | INT | DEFAULT 0 | 实践学时 |
| semester_id | INT | FOREIGN KEY | 开课学期ID，关联semesters表 |
| class_time | VARCHAR(100) | | 开课时间，格式：周X 第X-X节，如：周三 第 3-4 节 |
| classroom | VARCHAR(200) | | 上课地点，如：计算机学院 201 教室 |
| capacity | INT | DEFAULT 0 | 课程容量（可选人数） |
| selected_count | INT | DEFAULT 0 | 已选人数 |
| course_description | TEXT | | 课程描述 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**外键关系**：
- `semester_id` → `semesters.id` (ON DELETE SET NULL)

**索引**：
- `idx_course_no`：课程号索引
- `idx_course_name`：课程名称索引（支持模糊查询）
- `idx_course_type`：课程类型索引（支持筛选）
- `idx_semester_id`：学期ID索引

---

### 4. 课程教师关联表（course_teachers）

**用途**：存储课程与教师的关联关系（多对多关系）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 关联ID |
| course_id | INT | NOT NULL, FOREIGN KEY | 课程ID，关联courses表 |
| teacher_id | INT | NOT NULL, FOREIGN KEY | 教师ID，关联teachers表 |
| is_main_teacher | TINYINT(1) | DEFAULT 0 | 是否为主讲教师（0-否，1-是） |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**外键关系**：
- `course_id` → `courses.id` (ON DELETE CASCADE)
- `teacher_id` → `teachers.id` (ON DELETE CASCADE)

**唯一约束**：
- `uk_course_teacher`：同一课程同一教师只能关联一次

**索引**：
- `idx_course_id`：课程ID索引
- `idx_teacher_id`：教师ID索引

---

### 5. 学生选课关联表（student_courses）

**用途**：存储学生选课关系，**核心表**，用于已选课程展示

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | INT | PRIMARY KEY, AUTO_INCREMENT | 选课记录ID |
| student_no | VARCHAR(20) | NOT NULL | 学生学号，关联user表的student_no |
| course_id | INT | NOT NULL, FOREIGN KEY | 课程ID，关联courses表 |
| status | VARCHAR(20) | NOT NULL, DEFAULT '待修读' | 选课状态：待修读、已修读、已结课 |
| grade | DECIMAL(5,2) | | 成绩（已修读课程才有成绩） |
| select_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 选课时间 |
| create_time | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**外键关系**：
- `course_id` → `courses.id` (ON DELETE CASCADE)

**唯一约束**：
- `uk_student_course`：同一学生同一课程只能选一次

**索引**：
- `idx_student_no`：学生学号索引（**重要**：用于按学号筛选已选课程）
- `idx_course_id`：课程ID索引
- `idx_status`：选课状态索引（支持按状态筛选）

## 三、查询视图设计

### 视图：v_student_courses_detail

**用途**：已选课程详细信息视图，便于前端查询展示

**包含字段**：
- 选课信息：`select_id`, `student_no`, `select_status`, `grade`, `select_time`
- 课程信息：`course_id`, `course_no`, `course_name`, `course_type`, `credits`, `theory_hours`, `practice_hours`, `hours_display`（格式：理论学时/实践学时）
- 学期信息：`semester_name`, `semester_code`
- 上课信息：`class_time`, `classroom`
- 教师信息：`teachers`（多个教师用"，"分隔，主讲教师在前）
- 总学时：`total_hours`（理论学时 + 实践学时）

**使用示例**：
```sql
-- 查询学号为 'S2024001' 的学生的所有已选课程
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001'
ORDER BY select_time DESC;

-- 按学期筛选
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001' 
  AND semester_code = '2024-2025-1';

-- 按课程类型筛选
SELECT * FROM v_student_courses_detail 
WHERE student_no = 'S2024001' 
  AND course_type = '必修';
```

## 四、数据字典

### 课程类型（course_type）枚举值
- `必修`：必修课程
- `选修`：选修课程
- `专业核心`：专业核心课程
- `公共基础`：公共基础课程

### 选课状态（status）枚举值
- `待修读`：已选课但未开始修读
- `已修读`：正在修读中
- `已结课`：课程已结束

## 五、核心查询场景

### 1. 按学号查询已选课程列表
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = ? 
ORDER BY semester_code DESC, course_no ASC;
```

### 2. 按学期筛选
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = ? AND semester_code = ?;
```

### 3. 按课程类型筛选
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = ? AND course_type = ?;
```

### 4. 按课程名称/课程号模糊查询
```sql
SELECT * FROM v_student_courses_detail 
WHERE student_no = ? 
  AND (course_name LIKE ? OR course_no LIKE ?);
```

### 5. 统计已选课程总数
```sql
SELECT COUNT(*) AS total_courses 
FROM student_courses 
WHERE student_no = ?;
```

### 6. 统计已修学分总数
```sql
SELECT SUM(c.credits) AS total_credits
FROM student_courses sc
INNER JOIN courses c ON sc.course_id = c.id
WHERE sc.student_no = ? AND sc.status IN ('已修读', '已结课');
```

### 7. 统计平均学时
```sql
SELECT AVG(c.theory_hours + c.practice_hours) AS avg_hours
FROM student_courses sc
INNER JOIN courses c ON sc.course_id = c.id
WHERE sc.student_no = ?;
```

## 六、注意事项

1. **数据一致性**：
   - 删除课程时，会自动删除相关的选课记录（CASCADE）
   - 删除学期时，相关课程的学期ID会置为NULL（SET NULL）

2. **性能优化**：
   - 所有外键字段都建立了索引
   - 常用查询字段（student_no, course_type, status等）都建立了索引
   - 建议使用视图 `v_student_courses_detail` 进行查询，避免多表JOIN

3. **数据扩展**：
   - 如需支持更多课程类型，只需在 `course_type` 字段中添加新值
   - 如需支持更多选课状态，只需在 `status` 字段中添加新值

4. **与现有表的关系**：
   - `student_courses.student_no` 关联 `user.student_no`（需确保学号一致）
